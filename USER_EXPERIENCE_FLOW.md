## 详细流程说明

### 🏠 阶段 1: 首页入口

**页面**: `/`  
**用户目标**: 了解应用并进入系统

#### 用户看到的内容
- **视觉**: 
  - 黑色背景 (`#000000`)
  - 白色大标题 "SkiHIve"
  - 灰色副标题 "欢迎来到低空装配与仿真平台"
  - 两个按钮：白色"登录"按钮、透明边框"注册"按钮

#### 用户操作
- **点击"登录"** → 跳转到 `/login`
- **点击"注册"** → 跳转到 `/register`

#### 系统反馈
- 按钮悬停时透明度变化 (`hover:opacity-90`)
- 平滑的页面跳转

#### 情感体验
- **第一印象**: 简洁、专业、科技感
- **期待**: 对飞行体验的期待

### 🔐 阶段 2A: 登录流程

**页面**: `/login`  
**用户目标**: 使用已有账号登录

#### 用户看到的内容
- **顶部**: iOS 状态栏
- **Logo 区域**: SkiHIve Logo + "低空装配与仿真平台"
- **表单区域**:
  - 邮箱输入框（带占位符）
  - 密码输入框（带显示/隐藏切换）
  - "记住密码"复选框
  - "登录"按钮
  - "还没有账号？注册"链接

#### 用户操作流程
1. **输入邮箱** → 输入框获得焦点，显示光标
2. **输入密码** → 输入框获得焦点，可切换显示/隐藏
3. **勾选"记住密码"**（可选）→ 复选框状态切换
4. **点击"登录"按钮** → 
   - 如果未勾选协议 → 显示提示 "请先阅读并同意协议"
   - 如果已勾选 → 按钮显示加载状态（1秒）
   - 加载完成后 → 触发溶解动画（400ms）
   - 动画完成后 → 自动跳转到 `/flying`

#### 系统反馈
- **输入验证**: 实时验证邮箱格式
- **密码显示切换**: 点击眼睛图标切换显示/隐藏
- **加载状态**: 按钮显示加载动画
- **页面切换**: 溶解淡出效果（400ms）

#### 情感体验
- **流畅感**: 表单交互顺畅
- **安全感**: 密码隐藏保护隐私
- **期待感**: 登录成功后进入飞行体验

### ✈️ 阶段 3: 开始飞行体验 ⭐ 核心阶段

**页面**: `/flying`  
**用户目标**: 通过滑动启动飞行体验

#### 用户看到的内容

**初始状态**:
- **背景**: 
  - 黑色背景
  - `Group.png` 背景图位于 `y: 555px`（下方）
  - 微弱的紫色光辉效果（透明度 0.3）
- **文字内容**:
  - "Hi Jade！"（灰色，顶部）
  - "今天想要怎么飞？"（白色大标题，居中）
  - "低空装配与仿真平台"（灰色副标题）
  - "滑动开始飞行"（灰色提示文字）
- **滑动条**:
  - 半透明深灰色背景
  - 左侧白色圆形滑动按钮（带箭头图标）
  - 滑动条右侧有渐变填充提示

#### 用户操作流程

**步骤 1: 发现滑动交互**
- 用户看到"滑动开始飞行"提示
- 注意到左侧的滑动按钮

**步骤 2: 开始滑动**
- **操作**: 按住滑动按钮，向右拖拽
- **实时反馈**:
  - 滑动按钮跟随手指移动
  - 按钮轻微放大（`scale: 1.1`）
  - 背景 `Group.png` **开始向上滑动**（从 `y: 555px` 向 `y: -35px` 移动）
  - 光辉效果**逐渐增强**（透明度从 0.3 → 1，缩放从 1 → 1.2）
  - 滑动条背景填充**逐渐增加**（从 55px → 316.57px）
  - 所有动画实时响应，跟随滑动进度

**步骤 3: 滑动过程**
- **视觉变化**:
  - `Group.png` 持续向上移动（800ms 动画）
  - 光辉效果越来越明显
  - 滑动条填充越来越满
  - 紫色渐变从左侧向右扩展

**步骤 4: 完成滑动**
- **触发条件**: 滑动超过 80% (`maxDrag * 0.8`)
- **自动完成**:
  - 滑动按钮自动移动到最右侧（平滑动画）
  - `Group.png` 完成向上滑动到最终位置 `y: -35px`（800ms）
  - 光辉效果达到最大值
  - 滑动条完全填充

**步骤 5: 页面切换**
- **延迟触发**: 等待 800ms（让背景动画完成）
- **溶解动画**: 页面开始淡出（1200ms）
- **自动跳转**: 动画完成后跳转到 `/assembly`

#### 系统反馈

**实时响应**:
- 所有动画使用 `useMotionValue` 和 `useTransform` 实现实时响应
- 滑动进度直接映射到所有动画效果
- 无延迟，流畅自然

**视觉反馈**:
- 滑动按钮的缩放反馈
- 背景图的平滑移动
- 光辉效果的渐变增强
- 滑动条填充的进度显示

**完成反馈**:
- 自动完成滑动
- 平滑的溶解动画
- 无缝页面切换

#### 情感体验

**开始阶段**:
- **好奇**: "这个滑动条是做什么的？"
- **探索**: 尝试滑动，发现背景在移动

**滑动过程**:
- **兴奋**: 看到背景向上滑动，光辉增强
- **掌控感**: 通过滑动控制动画进度
- **期待**: 滑动越多，期待感越强

**完成阶段**:
- **满足感**: 成功完成滑动
- **成就感**: 看到完整的动画效果
- **期待**: 对下一个页面的期待

#### 关键交互细节

1. **滑动约束**: 
   - 只能水平滑动（`drag="x"`）
   - 滑动范围：0 到 261px
   - 有弹性效果（`dragElastic: 0.1`）

2. **动画映射**:
   ```
   滑动进度 (0 → 261px)
     ↓
   Group.png Y 位置 (555px → -35px)
  光辉透明度 (0.3 → 1)
  光辉缩放 (1 → 1.2)
  填充宽度 (55px → 316.57px)
   ```

3. **完成判断**:
   - 滑动超过 80% 自动完成
   - 未达到 80% 自动回弹到起始位置

**用户目标**: 选择飞行器组件并完成装配

  - 底层：`Group 20.svg` 静态背景（无交互）
  - 中心：显示当前选中的组件或装配状态
- **周围**: 
  - 6 个组件按钮（54.29px × 54.29px）：
    - 机架（顶部居中，已选中）
    - 螺旋桨（左上）
    - 电机（右上）
    - 电池（左下）
    - 传感器（右下）

  - AI 分析状态：`Group 68.svg`（带流动光效边框，8秒循环）

- 用户看到 6 个组件按钮

**操作 2: 选择组件**

**操作 3: 查看 3D 预览**

**操作 4: 观察 AI 分析**

**操作 5: 完成装配**
- **选中所有组件后**:
  - 进度显示 "6/6"
  - 3D 预览显示"装配完成！"
  - "下一步"按钮变为激活状态
- **点击"下一步，场景仿真"** → 跳转到 `/environment`

**操作 6: 重置装配**（可选）
- **点击重置按钮** →

## 交互细节

### 动画时长标准

| 动画类型 | 时长 | 用途 |
|---------|------|------|
| 快速反馈 | 0.3s - 0.5s | 按钮点击、悬停效果 |
| 标准过渡 | 0.8s - 1.2s | 页面切换、状态变化 |
| 慢速动画 | 2s - 8s | 装饰性动画、流动效果 |

### 交互反馈层级

1. **即时反馈** (0ms - 100ms)
   - 按钮按下效果
   - 输入框焦点
   - 滑动跟随

2. **短时反馈** (100ms - 500ms)
   - 选中状态变化
   - 小范围动画
   - 状态指示器

3. **中时反馈** (500ms - 1200ms)
   - 页面切换
   - 组件装配
   - 完成动画

4. **长时反馈** (1200ms+)
   - 背景动画
   - 装饰性效果
   - 流动光效

### 视觉层次

1. **背景层** (z-index: 0)
   - 静态背景图
   - 装饰性元素

2. **内容层** (z-index: 10)
   - 主要交互元素
   - 文字内容
   - 组件图标

3. **覆盖层** (z-index: 20+)
   - 状态栏
   - 弹窗
   - 加载遮罩

## 情感体验设计

### 情感曲线

```
期待 → 探索 → 兴奋 → 满足 → 成就感 → 期待
```

## 关键决策点

### 用户决策路径

```
开始飞行
  ↓
[是否完成滑动？]
  ├─ 是 → 进入装配页面
  └─ 否 → 继续滑动或放弃

组件装配
  ↓
[选择哪个组件？]
  ├─ 点击组件 → 进入选择页面
  └─ 继续浏览 → 查看其他组件

组件选择
  ↓
[选择哪个型号？]
  ├─ 确认选择 → 返回装配页面
  └─ 返回 → 取消选择

装配完成
  ↓
[是否继续？]
  ├─ 是 → 进入场景选择
  └─ 否 → 重置装配

场景选择
  ↓
[选择哪个场景？]
  ├─ 确认选择 → 开始仿真
  └─ 返回 → 取消选择
```

### 关键交互点

1. **滑动完成判断** (`/flying`)
   - 阈值：80% 滑动距离
   - 反馈：自动完成 + 动画
   - 容错：未达到自动回弹

2. **组件选择确认** (`/component-selector`)
   - 操作：点击确认按钮
   - 反馈：返回 + 状态更新
   - 容错：返回按钮取消

3. **装配完成判断** (`/assembly`)
   - 条件：选中 6 个组件
   - 反馈：完成提示 + 按钮激活
   - 容错：重置按钮

## 总结

SkiHIve 的用户体验流程设计注重：

1. **流畅性**: 所有页面切换和动画都经过精心设计，确保流畅自然
2. **交互性**: 核心页面（飞行、装配）提供丰富的交互体验
3. **反馈性**: 每个操作都有明确的视觉和交互反馈
4. **情感性**: 通过动画和视觉效果强化用户的情感体验
5. **引导性**: 清晰的流程和提示引导用户完成操作

整个流程从简单的入口页面，到核心的交互体验，再到最终的场景选择，形成了一个完整的用户体验闭环。
